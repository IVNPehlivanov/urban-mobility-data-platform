# Shared Bicycle Data Platform

A **data engineering and analytics project** that models and analyzes
shared mobility systems (bicycles, scooters, e-bikes) using **Apache
Spark**, **Hadoop**, and **Python**.

This project demonstrates distributed data processing, pipeline
orchestration, and data-driven insights for smart city mobility systems.

------------------------------------------------------------------------

##  Tech Stack

-   **Python 3.11**
-   **Apache Spark**, **Hadoop**
-   **PyCharm**, **Jupyter Notebook**
-   **Git / GitHub / Gitlab**

------------------------------------------------------------------------

##  Key Features

-   Distributed data processing with Spark
-   NoSQL + relational data integration
-   Ride and station utilization analytics
-   Maintenance and subscription insights
-   Scalable ETL pipeline setup

------------------------------------------------------------------------

##  Setup Instructions

1.  **Install Python 3.11** →
    [python.org](https://www.python.org/downloads/release/python-3119/)

2.  **Install Java 8--17** (required by Spark)

3.  Clone the repo:

    ``` bash
    git clone https://github.com/IVNPehlivanov/urban-mobility-data-platform.git
    cd urban-mobility-data-platform
    pip install -r requirements.txt
    ```

4.  Configure Spark and Hadoop paths in `ConnectionConfig.py`

5.  Run initial tests with:

    ``` bash
    jupyter notebook CHECK_1_install.ipynb
    ```
